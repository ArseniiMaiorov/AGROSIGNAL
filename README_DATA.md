## Термины и сокращения

| Термин / Сокращение | Определение |
|---|---|
| **АОИ** | Область интереса (Area of Interest) — географический регион, в пределах которого выполняется обработка спутниковых данных. |
| **BSI** | Индекс голого почвенного покрова (Bare Soil Index) — спектральный индекс, рассчитываемый по каналам Sentinel-2. |
| **ГРД** | Геопространственные растровые данные — данные, хранящие значения пикселей в привязке к географическим координатам. |
| **NDMI** | Нормализованный индекс влажности (Normalized Difference Moisture Index). |
| **NDVI** | Нормализованный разностный вегетационный индекс (Normalized Difference Vegetation Index). |
| **NDWI** | Нормализованный разностный водный индекс (Normalized Difference Water Index). |
| **MNDWI** | Модифицированный нормализованный разностный водный индекс (Modified NDWI). |
| **NPZ** | Формат хранения массивов NumPy (NumPy Zipped Archive). |
| **ONNX** | Открытый формат нейронных сетей (Open Neural Network Exchange). |
| **ПО** | Программное обеспечение. |
| **SAM** | Segment Anything Model — нейросетевая модель универсальной сегментации объектов (Meta AI). |
| **SCL** | Scene Classification Layer — растр классификации сцены Sentinel-2 (1–11 классов). |
| **Sentinel-2** | Спутниковая миссия ЕКА для наблюдения Земли с пространственным разрешением 10–60 м. |
| **TIF / GeoTIFF** | Формат географически привязанных растровых изображений. |
| **UNet** | Архитектура полностью свёрточной нейронной сети типа «энкодер–декодер» с пропускными соединениями, предназначенная для семантической сегментации. |

## 1 Общие сведения

### 1.1 Назначение набора данных

Набор данных программного комплекса AGROSIGNAL предназначен для обучения и верификации моделей автоматического выделения границ сельскохозяйственных полей по многоспектральным снимкам спутника Sentinel-2. В состав набора входят предобработанные тайлы спутниковых данных, метки полей, обученные нейросетевые модели и вспомогательные конфигурационные файлы.

Система реализует следующие функции обработки данных:

- автоматическое обнаружение и оцифровку границ сельскохозяйственных угодий по мультиспектральным снимкам Sentinel-2;
- постфильтрацию сегментированных объектов с применением ансамблевого классификатора;
- региональное дообучение основной модели сегментации на данных конкретных субъектов Российской Федерации.

## 3 Спутниковые данные Sentinel-2

### 3.1 Характеристики тайлов

Тайл представляет собой предобработанный фрагмент многоспектрального снимка Sentinel-2, хранимый в формате NPZ. Каждый тайл охватывает область 512×512 пикселей при пространственном разрешении 10 м/пиксель, что соответствует площади приблизительно 26,2 км². Для получения данных используются спектральные каналы B2 (490 нм), B3 (560 нм), B4 (665 нм), B8 (842 нм), B11 (1610 нм), B12 (2190 нм) и слой классификации сцены SCL.

### 3.2 Структура NPZ-файла

Каждый файл формата `.npz` содержит набор именованных массивов, перечисленных в таблице 4.

**Таблица 4 — Состав NPZ-файла тайла Sentinel-2**

| Ключ | Форма массива | Тип данных | Содержание |
|---|---|---|---|
| `features` | (16, 512, 512) | float32 | Стек вычисленных признаков (16 каналов) |
| `B2` | (512, 512) | float32 | Отражение в синем канале |
| `B3` | (512, 512) | float32 | Отражение в зелёном канале |
| `B4` | (512, 512) | float32 | Отражение в красном канале |
| `B8` | (512, 512) | float32 | Отражение в ближнем ИК-диапазоне |
| `B11` | (512, 512) | float32 | Отражение в SWIR-1 диапазоне |
| `B12` | (512, 512) | float32 | Отражение в SWIR-2 диапазоне |

## 4 Метки полей

### 4.1 Формат хранения

Метки полей хранятся в формате GeoTIFF (один растровый канал, тип данных uint8) в координатной привязке EPSG:4326, согласованной с соответствующим NPZ-тайлом. Значения пикселей кодируются следующим образом:

- значение `0` — фон (отсутствие сельскохозяйственного поля);
- значение `1` — сельскохозяйственное поле (слабая метка).

### 4.2 Метод формирования слабых меток

Формирование меток выполняется скриптом `backend/training/generate_weak_labels_real_tiles.py` посредством следующей последовательности операций:

1. Запрос к Overpass API (OpenStreetMap) для получения полигонов с тегами `landuse=farmland`, `landuse=meadow`, `landuse=orchard` в пределах ограничивающего прямоугольника тайла.
2. Растеризация полученных полигонов в пиксельную сетку тайла (EPSG:4326).
3. Морфологическая постобработка: дилатация и эрозия для устранения краевых артефактов.

### 4.3 Характеристики покрытия меток

Степень покрытия меток существенно варьируется в зависимости от региона и определяется плотностью разметки в OpenStreetMap. В таблице 10 приведены типичные диапазоны покрытия.

**Таблица 10 — Типичное покрытие меток по регионам**

| Регион | Диапазон покрытия, % | Причина вариации |
|---|---|---|
| Краснодарский край | 15–65 | Высокая плотность и качество разметки OSM |
| Пермский край | 1–20 | Мелкоконтурные поля, ограниченная разметка OSM |
| Глобальный датасет | 5–55 | Зависит от страны и региона |

## 5 Описание моделей

### 5.1 Модель сегментации BoundaryUNet v3

#### 5.1.1 Архитектура

Основная модель сегментации реализована на основе архитектуры UNet с энкодером и декодером, связанными пропускными соединениями. Модель принимает на вход стек из 16 спектральных признаков и формирует три выходных карты вероятностей.

Структура архитектуры:

```
Энкодер:
  Блок conv0:   16 → 32 канала,  stride 1  (свёртка 3×3, BatchNorm, ReLU)
  Блок conv1:   32 → 64 канала,  stride 2  (MaxPool + двойная свёртка)
  Блок conv2:   64 → 128 каналов, stride 2
  Блок conv3:  128 → 256 каналов, stride 2

Бутылочное горлышко:
  Блок conv4:  256 → 512 каналов  (Dropout p = 0,30)

Декодер:
  Блок up3:  512 + 256 → 256 каналов  (Dropout p = 0,20)
  Блок up2:  256 + 128 → 128 каналов  (Dropout p = 0,10)
  Блок up1:  128 + 64  → 64 канала
  Блок up0:   64 + 32  → 32 канала

Выходной блок:
  out: 32 → 3 канала (функция активации Sigmoid)
```

Три выходных канала модели:
- **Канал 0 — Extent**: вероятность принадлежности пикселя к площади поля;
- **Канал 1 — Boundary**: вероятность принадлежности пикселя к границе поля;
- **Канал 2 — Distance**: нормализованное значение расстояния до ближайшей границы поля.

Общее число обучаемых параметров составляет **7 766 851**.

#### 5.1.2 Файлы модели

**Таблица 11 — Файлы модели BoundaryUNet v3**

| Файл | Размер | Назначение |
|---|---|---|
| `boundary_unet_v3_cpu.pth` | ~30 МБ | Чекпойнт модели в формате PyTorch |
| `boundary_unet_v3_cpu.onnx` | ~30 МБ | Экспорт модели в формате ONNX (opset 18) |
| `boundary_unet_v3_cpu.norm.json` | <1 МБ | Статистики нормализации признаков |

#### 5.1.3 Параметры нормализации входных данных

Перед подачей на вход модели признаки нормализуются по формуле z-нормализации с параметрами, хранящимися в файле `boundary_unet_v3_cpu.norm.json`. Параметры по каждому каналу приведены в таблице 5 (раздел 3.3).

Дополнительные параметры конфигурации модели:
- порог классификации по каналу Extent: **0,42**;
- версия ONNX opset: **18**;
- коэффициент Dropout на уровне бутылочного горлышка: **0,30**;
- коэффициент Dropout на уровне блока up3: **0,20**;
- коэффициент Dropout на уровне блока up2: **0,10**.

#### 5.1.4 Параметры обучения

Обучение модели выполнено со следующими параметрами:

- функция потерь: комбинированная (BCE + Dice для канала Extent; BCE для канала Boundary; MSE для канала Distance);
- оптимизатор: AdamW;
- планировщик скорости обучения: CosineAnnealingLR;
- аугментация данных: случайное отражение, поворот, упругая деформация, случайное кадрирование.

### 5.2 Классификатор объектов

#### 5.2.1 Описание

Классификатор объектов предназначен для постфильтрации сегментов, выделенных моделью BoundaryUNet. Классификатор принимает на вход вектор из 19 признаков, описывающих геометрические и спектральные характеристики сегмента, и выдаёт оценку вероятности того, что объект является сельскохозяйственным полем.

Тип модели: `sklearn.ensemble.HistGradientBoostingClassifier`.
Файл: `backend/models/object_classifier.pkl`.
Порог классификации (параметр `OBJECT_MIN_SCORE`): **0,40**.

### 8.4 Запуск пайплайна

Полный запуск пайплайна для обоих регионов выполняется следующей командой:

```bash
PYTHONPATH=backend python backend/training/train_regional_supplement.py \
    --regions krasnodar perm \
    --epochs 50 \
    --push-gdrive
```

При необходимости пропуска уже выполненных этапов используются флаги `--skip-fetch` и `--skip-labels`:

```bash
PYTHONPATH=backend python backend/training/train_regional_supplement.py \
    --regions krasnodar perm \
    --epochs 50 \
    --skip-fetch \
    --skip-labels
```

По завершении пайплайна обновляются файлы `backend/models/boundary_unet_v3_cpu.pth`, `backend/models/boundary_unet_v3_cpu.onnx` и формируется отчёт `backend/models/regional_training_report_YYYYMMDD_HHMMSS.json`.
