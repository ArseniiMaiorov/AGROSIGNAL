## 2. Техническая оценка качества
### Сильные стороны:
- **Архитектура**: Четкое разделение на микросервисы (API, Workers, Redis, DB). Использование FastAPI и Celery обеспечивает надежную обработку тяжелых ГИС-задач.
- **Дизайн**: Исключительная эстетичность интерфейса («ретро-футуристический киберпанк»). Продуманное UX с панелями мониторинга и журналом событий.
- **Стабильность**: Реализована система префлайт-чеков (preflight checks) для оценки сложности задачи перед запуском.
- **Безопасность**: Реализован полноценный RBAC (Role-Based Access Control) с поддержкой организаций (Tenancy). Пароли хешируются с использованием `scrypt`.
- **Логирование**: Глубокое структурированное логирование (JSON) на бэкенде позволяет быстро диагностировать проблемы (например, таймауты внешних API).

### Уровень реализации:
Проект находится на этапе **высокой готовности**. Реализованы не только базовые функции визуализации, но и сложные AI-пайплайны обработки спутниковых снимков с системой отказоустойчивости (fallback на эвристики при отсутствии моделей).

## 3. Рекомендации по улучшению
1. **Frontend Performance**: При большом количестве полей (>1000) Leaflet может начать замедляться. Рекомендуется переход на WebGL-рендеринг или использование векторных тайлов (MVT).
2. **External API Robustness**: Замечены таймауты от CDS API (Copernicus). Стоит внедрить более агрессивное кэширование архивных погодных данных и фоновый префетч для активных регионов.
3. **UX/UI**:
    - Добавить возможность массового редактирования полей (например, смена культуры для всей группы).
    - Реализовать экспорт отчетов в PDF с графиками динамики NDVI.
4. **DevOps**: Конфигурация Docker Compose требует значительного времени на сборку `npm install`. Рекомендуется использовать многоэтапную сборку (multi-stage builds) с сохранением слоев кэша.

### Глубокий анализ проблем (О результатах стресс-теста)
1. **Баг интерактивности карты (выделение полей)**: 
   Анализ кода (`MapView.vue` и `useFieldPanelData.js`) и данных БД выявил, что функции выборки `map.forEachFeatureAtPixel` и `store.selectField` работают корректно. Проблема возникает из-за того, что если процесс детекции не дошел до конца на UI (остановка на 15.85%), функция `fetchResult()` не вызывается, и интерактивные векторные поля (`fieldsGeoJson`) не загружаются на карту. Пользователь видит только фоновую WMS-подложку (растр) от Sentinel, которая некликабельна.
2. **Рассинхрон логов и статус 15.85%**: 
   Опрос статуса (`pollStatus` в `store/map.js`) имеет жесткий exponential backoff (до 60 секунд) в случае сетевой задержки. Сервисы Celery из-за тяжелой ML модели могут отвечать с задержкой, что заставляет фронтенд замораживать интерфейс на 15.85% на долгую секунду, даже если сам бекенд (как показано в таблице `aoi_runs`) успешно и быстро завершает работу со статусом `done` и `100%`.
3. **Перекрытие интерфейса (Floating Windows)**: 
   В `DraggableWindow.vue` нет защиты от закрытия всей области просмотра на малых экранах. Окна имеют жесткие размеры, что приводит к блокировке кликов по холсту. Необходима реализация кнопок "Свернуть все" или автоматического тайлинга (auto-tiling).
4. **Бекенд Celery**: 
   Падение `cleanup_zombie_runs` связано с конфликтом циклов событий `asyncio` и пулов соединений в `asyncpg` во время фоновой очистки зомби-процессов.
